# Changelog

## v1

Initial release, aligned with ModuLab Core specification chapter 4 (v1 API). Includes the required fields name, version, author, license, category, min_core_api, min_core_ui, tier and scope, plus the optional fields startup, crud, handler, resources, depends_on, storage, credentials (with per_location), egress_allowlist, requested_scopes and jobs (with catch_up). Additional fields for modulab-community discovery entries: source_repo, manifest_path and release_url.
